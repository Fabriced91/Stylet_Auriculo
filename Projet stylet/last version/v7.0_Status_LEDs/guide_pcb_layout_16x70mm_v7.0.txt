DIMENSIONS PCB :
  • Largeur : 16mm (max absolu, préférer 15mm pour jeu mécanique)
  • Longueur : 62mm (RÉDUIT de 70mm v6 - simplification circuit LED)
  • Épaisseur : 1.6mm standard (ou 1.0mm pour plus de flexibilité)
  • Forme : Rectangle pur (pas d'encoches, simplifie fabrication)

TUBE STYLET :
  • Diamètre intérieur : 18mm
  • Jeu mécanique PCB/tube : 1mm de chaque côté (16mm PCB dans Ø18mm)
  • Matériau : Aluminium anodisé (dissipation thermique LED)

MONTAGE :
  • PCB inséré verticalement dans le tube
  • LED NSPW500CS 5mm MONTÉE DIRECTEMENT SUR PCB (through-hole)
  • Trous de fixation M2 aux 2 extrémités (écartement 57mm)
  • Boutons latéraux traversant le tube (trous Ø4mm)

NOUVEAUTÉS v7.0 :
  ✓ PCB 8mm plus court (62mm vs 70mm)
  ✓ Suppression zone driver LED (7 composants éliminés)
  ✓ LED through-hole directe (pas de MCPCB externe)
  ✓ Montage simplifié (moins de soudures)

 📊 RÈGLES DE CONCEPTION (Design Rules)
═══════════════════════════════════════════════════════════════════════════════

LARGEUR DES TRACES :
  • Signal digital (I2C, boutons) : 0.2mm (8 mil)
  • Alimentation 3.3V : 0.3mm (12 mil)
  • PWM LED (20mA) : 0.25mm (10 mil) ← RÉDUIT car faible courant
  • BAT+ (jusqu'à 250mA max) : 0.4mm (16 mil) ← RÉDUIT car faible conso
  • LED principale (20mA) : 0.25mm (10 mil) ← RÉDUIT de 0.6mm v6
  • GND stitching via : Ø0.3mm pad Ø0.6mm

CLEARANCE (Espacement) :
  • Trace-to-trace : 0.15mm (6 mil) minimum
  • Trace-to-pad : 0.15mm
  • Pad-to-pad (composants différents) : 0.2mm
  • USB-C differential pair : 0.15mm gap, 90Ω impédance

VIA :
  • Diamètre perçage : 0.3mm (12 mil)
  • Diamètre pad : 0.6mm (24 mil)
  • Via stitching GND : Espacement 2-3mm sur plan GND

PLANS DE CUIVRE :
  • Layer 1 (Top) : Signaux + composants
  • Layer 2 (Bottom) : Plan GND continu (> 80% surface)
  • Clearance plan/trace : 0.2mm
  • Thermal relief sur pads : 4 spokes, 0.3mm largeur

SOUDURE :
  • Masque soudure (solder mask) : Vert
  • Sérigraphie (silkscreen) : Blanc
  • Finition : ENIG (or chimique) ou HASL (étain-plomb)

 🔌 ROUTAGE CRITIQUE (Signaux sensibles)
═══════════════════════════════════════════════════════════════════════════════

1️⃣ PWM LED (ATmega328P PD3 → R_LED → LED+)
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   Contraintes :
     • Trace DIRECTE < 15mm (minimiser parasites PWM)
     • Largeur 0.25mm suffisante (20mA max)
     • Éloignée de l'ADC (PC0) > 5mm
     • Résistance 47Ω AU PLUS PRÈS de la LED+ (série)
   
   Routage recommandé :
     ATmega328P PD3 (pin 5) → [Trace 0.25mm] → [R_LED 47Ω 0805] → LED+ (anode)
     LED- (cathode) → GND (via through-hole pad, court chemin vers plan GND)

   SIMPLIFICATION v7.0 :
     ✓ Plus d'op-amp ni MOSFET → Routage ultra-simple
     ✓ Pas de traces haute puissance (350mA → 20mA)
     ✓ Pas de filtrage RC complexe
     ✓ 1 trace, 1 résistance, c'est tout !

2️⃣ ADC Batterie (Diviseur → ATmega328P PC0/ADC0)
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   Contraintes :
     • Filtrage capacitif 100nF au plus près de PC0
     • Éloignée du PWM et switching (LDO, TP4056)
     • Résistances précises 1% (si disponible)
   
   Routage :
     BAT+ → [R2 100k] → [Nœud] → [R3 33k] → GND
                          ↓
                     [C 100nF] → GND
                          ↓
                    ATmega328P PC0 (pin 23)

3️⃣ I2C Bus (SDA/SCL vers PCF8574)
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   Contraintes :
     • Longueur < 50mm (vitesse standard 100kHz OK)
     • Pull-ups 4.7kΩ sur SDA ET SCL (près du master ATmega)
     • Condensateur 100nF sur VCC PCF8574
   
   Routage :
     ATmega328P PC4 (SDA) ───[Trace]───┬─── PCF8574 SDA
                                       │
                                  [4.7kΩ] → 3.3V
     
     ATmega328P PC5 (SCL) ───[Trace]───┬─── PCF8574 SCL
                                       │
                                  [4.7kΩ] → 3.3V

4️⃣ Alimentation VCC (LDO → ATmega328P)
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   Contraintes :
     • Traces larges 0.3mm minimum (distribution 3.3V)
     • Condensateurs découplage 100nF < 3mm des pins VCC
     • Ferrite bead entre VCC et AVCC (filtrage ADC)
   
   Routage :
     HT7333 OUT → [Trace 0.3mm] → ATmega VCC (pin 7)
                               ↓
                          [C1 100nF] → GND
     
     VCC → [FB1 Ferrite] → AVCC (pin 20)
                        ↓
                   [C2 100nF] → GND

 ⚡ GESTION THERMIQUE
═══════════════════════════════════════════════════════════════════════════════

COMPOSANTS CHAUDS (v7.0 - Dissipation minimale) :

  1. LED NSPW500CS (5mm through-hole) : P = Vf × If = 3.3V × 0.02A = 0.066W
     → Dissipation NÉGLIGEABLE (vs 1.05W LED 519A v6)
     → Pas de dissipateur requis
     → Pad thermique optionnel : Zone copper autour pads (5×5mm)
     → Tube aluminium suffit amplement

  2. Résistance 47Ω (0805) : P = I²R = 0.020² × 47 = 0.019W
     → Totalement négligeable (SMD 0805 = 125mW max)
     → Pas de précautions thermiques

  3. LDO HT7333-A : P = (Vin - Vout) × Iout
     → P = (4.2 - 3.3) × 0.015 = 0.0135W (négligeable, 15mA conso totale)
     → Pas de dissipateur requis

⚠️ COMPARAISON v6 → v7 :
   v6 : Rsense 4.7Ω 1W → 0.58W dissipée (chaud !)
   v7 : R_LED 47Ω 0805 → 0.019W dissipée (froid)
   
   v6 : MOSFET → 0.19W (chaud)
   v7 : Pas de MOSFET → 0W
   
   v6 : LED 519A → 1.05W (très chaud, MCPCB requis)
   v7 : LED NSPW500CS → 0.066W (tiède)
   
   GAIN : -1.73W dissipation totale (-96% !)

DISSIPATION TUBE ALUMINIUM :
  Le tube métallique Ø18mm sert de dissipateur naturel (largement surdimensionné).
  • Contact thermique PCB/tube via pads GND exposés (optionnel)
  • Pas de graisse thermique nécessaire
  • LED peut toucher le tube en bout sans risque de surchauffe

ÉTAPE 1 : SCHÉMATIQUE (Eeschema)
  ☐ 1.1 Créer nouveau projet "stylet_auriculo_v7.0"
  ☐ 1.2 Importer librairies composants (créer footprints custom si besoin)
  ☐ 1.3 Dessiner schéma selon "schema_electronique_ATmega328P_v7.0.txt"
  ☐ 1.4 SUPPRIMER composants v6 : MCP6001, 2N7002, Rsense, R_GATE, filtres RC
  ☐ 1.5 AJOUTER LED NSPW500CS (5mm through-hole) + R_LED 47Ω
  ☐ 1.6 Annoter composants (Tools → Annotate Schematic)
  ☐ 1.7 Assigner footprints :
        • LED1 : LED_THT:LED_D5.0mm (standard 5mm)
        • R1 : Resistor_SMD:R_0805_2012Metric
  ☐ 1.8 Générer netlist (Tools → Generate Netlist)
  ☐ 1.9 ERC check (Inspect → Electrical Rules Checker)
  ☐ 1.10 Exporter PDF schéma pour documentation

ÉTAPE 2 : PCB LAYOUT (Pcbnew)
  ☐ 2.1 Importer netlist (Tools → Update PCB from Schematic)
  ☐ 2.2 Définir edge cuts (rectangle 16×62mm) ← RÉDUIT de 70mm
  ☐ 2.3 Placer composants selon plan ci-dessus
  ☐ 2.4 Ajouter trous fixation M2 (écartement 57mm)
  ☐ 2.5 Créer plan GND layer bottom (Add Filled Zone)
  ☐ 2.6 Router PWM LED (PD3 → R_LED → LED+) manuellement
  ☐ 2.7 Router signaux critiques (ADC, I2C) manuellement
  ☐ 2.8 Router signaux standard (autorouter si désiré)
  ☐ 2.9 Ajouter via stitching GND
  ☐ 2.10 Ajouter zone copper optionnelle sous LED (dissipation)
  ☐ 2.11 Annotations silkscreen (textes, polarités, v7.0 marking)
  ☐ 2.12 DRC check (Inspect → Design Rules Checker)
  ☐ 2.13 3D View (View → 3D Viewer) pour vérification mécanique
  ☐ 2.14 Vérifier hauteur LED (surélevée 2mm pour alignement optique)

ÉTAPE 3 : FABRICATION
  ☐ 3.1 Générer Gerbers (File → Plot → Gerber format)
      → Layers : F.Cu, B.Cu, F.SilkS, B.SilkS, F.Mask, B.Mask, Edge.Cuts
  ☐ 3.2 Générer Drill file (Generate Drill Files → PTH & NPTH)
  ☐ 3.3 Zipper tous les Gerbers
  ☐ 3.4 Exporter BOM (File → Fabrication Outputs → BOM) selon BOM_complete_v7.0.txt
  ☐ 3.5 Exporter Position file pour assemblage (CPL format)
  ☐ 3.6 Uploader sur JLCPCB/PCBWay
  ☐ 3.7 Vérifier preview Gerber en ligne
  ☐ 3.8 Commander (options : 2L, 1.6mm, ENIG, vert)
