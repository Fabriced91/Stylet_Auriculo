═══ ÉTAPE 2 : LED PRINCIPALE (Version ultra-simple v7.0) ═══

MONTAGE DIRECT PWM → RÉSISTANCE → LED :

```
Arduino D3 (PWM) ──[47Ω]── Anode LED NSPW500CS (+, patte longue)
                              │
                          Cathode LED (-, patte courte)
                              │
                             GND
```

⚡ NOUVEAUTÉ v7.0 : MONTAGE ULTRA-SIMPLIFIÉ !

Fini les drivers complexes ! La LED NSPW500CS consomme seulement 20mA,
donc le microcontrôleur peut la piloter DIRECTEMENT via une résistance
de limitation de courant.

Composants :
  • LED NSPW500CS (5mm through-hole) :
    - Anode (+, patte longue) → Résistance 47Ω → Arduino D3
    - Cathode (-, patte courte) → GND

  • Résistance 47Ω (1/4W traversante) :
    Limite courant LED à ~20mA @ 3.3V (PWM 100%)
    Calcul : I = (Vcc - Vf) / R = (5V - 3.3V) / 47Ω ≈ 36mA max
             Avec PWM, courant moyen = 36mA × duty_cycle
             Pour 20mA → duty cycle = 20/36 = 56%

2.1 - Insérer LED NSPW500CS sur breadboard (espacer pattes 0.3" / 7.62mm)
2.2 - Identifier polarité : patte longue = anode (+), patte courte = cathode (-)
2.3 - Connecter cathode (patte courte) à rail GND
2.4 - Connecter anode (patte longue) à résistance 47Ω
2.5 - Connecter résistance 47Ω à Arduino D3

VÉRIFICATION :
  • Avec multimètre en mode continuité :
    - Cathode LED → GND : court-circuit (OK)
    - Anode → D3 (via 47Ω) : résistance mesurée ~47Ω (OK)
  
  • Test rapide :
    - Upload code test : digitalWrite(3, HIGH);
    - LED doit s'allumer à intensité modérée (~36mA)
    - Si trop brillante, augmenter résistance (82Ω pour 15mA)

⚠️ DIFFÉRENCE v6 → v7 :

  v6 (350mA LED) : Nécessitait driver complexe

 💻 CODE ARDUINO UNO v7.0 (Version Test Breadboard)
═══════════════════════════════════════════════════════════════════════════════

Voici le code adapté pour Arduino Uno avec LED NSPW500CS 20mA :

```cpp
/*
 * STYLET AURICULOTHÉRAPIE v7.0 - Test Breadboard Arduino Uno
 * 
 * LED NSPW500CS 20mA - Circuit simplifié (résistance 47Ω)
 * Pins mapping :
 * - D3  : PWM LED principale (Timer2, OC2B) - NOUVEAU v7.0
 * - D11, D10, D9 : LED indicateurs R, G, B
 * - D2 : Bouton FREQ
 * - D4 : Bouton MODE (changé de D3 v6)
 * - A0 : ADC mesure batterie
 */

// ═══ CONFIGURATION PINS ═══
#define LED_PWM_PIN     3     // PWM LED principale (NOUVEAU v7.0 : D3 au lieu de D11)
#define LED_R_PIN       11    // LED Rouge (indicateur)
#define LED_G_PIN       10    // LED Verte
#define LED_B_PIN       9     // LED Bleue
#define BTN_FREQ_PIN    2     // Bouton FREQ
#define BTN_MODE_PIN    4     // Bouton MODE (changé de D3)
#define ADC_BATTERY_PIN A0    // Mesure batterie

// ═══ FRÉQUENCES DE NOGIER ═══
const float frequencies[] = {2.28, 4.56, 9.12, 18.25, 36.50, 73.0, 146.0};
const char freqNames[] = {'A', 'B', 'C', 'D', 'E', 'F', 'G'};
const uint8_t numFreqs = 7;

// ═══ MODES ═══
enum Mode {MODE_DETECTION, MODE_TRAITEMENT, MODE_BLANC};
const char* modeNames[] = {"DETECTION", "TRAITEMENT", "BLANC"};
uint8_t numModes = 3;

// ═══ DUTY CYCLES v7.0 (adaptés pour LED 20mA) ═══
#define DUTY_DETECTION   64   // 25% → ~5mA
#define DUTY_TRAITEMENT  235  // 92% → ~20mA
#define DUTY_BLANC       128  // 50% → ~10mA

// ═══ INDICATEURS LED RGB ═══
const bool freqIndicators[7][3] = {
  {1, 0, 0},  // A : Rouge
  {0, 1, 0},  // B : Vert
  {1, 1, 0},  // C : Jaune (R+G)
  {0, 0, 1},  // D : Bleu
  {1, 0, 1},  // E : Magenta (R+B)
  {0, 1, 1},  // F : Cyan (G+B)
  {1, 1, 1}   // G : Blanc (R+G+B)
};

// ═══ VARIABLES GLOBALES ═══
uint8_t currentFreq = 0;
Mode currentMode = MODE_DETECTION;

void setup() {
  
  // Configuration pins
  pinMode(LED_PWM_PIN, OUTPUT);
  pinMode(LED_R_PIN, OUTPUT);
  pinMode(LED_G_PIN, OUTPUT);
  pinMode(LED_B_PIN, OUTPUT);
  pinMode(BTN_FREQ_PIN, INPUT_PULLUP);
  pinMode(BTN_MODE_PIN, INPUT_PULLUP);
  
  // Configuration Timer2 pour PWM sur D3 (OC2B)
  setupTimer2PWM();
  
  // Serial pour debug
  Serial.begin(115200);
  Serial.println(F("\n╔════════════════════════════════════╗"));
  Serial.println(F("║  STYLET AURICULOTHÉRAPIE v7.0      ║"));
  Serial.println(F("║  Test Breadboard - Arduino Uno     ║"));
  Serial.println(F("║  LED NSPW500CS 20mA - Simplifié    ║"));
  Serial.println(F("╚════════════════════════════════════╝\n"));
  
  // Initialisation indicateurs
  updateFreqIndicators();
  printStatus();
  
  // Test LED principale (3 flashs)
  for (uint8_t i = 0; i < 3; i++) {
    OCR2B = 255;  // NOUVEAU : OCR2B au lieu de OCR2A
    delay(100);
    OCR2B = 0;
    delay(100);
  }
}

void setupTimer2PWM() {
  // Timer2 Fast PWM sur OC2B (pin D3), fréquence ~31kHz
  // NOUVEAU v7.0 : Utilisation de OC2B au lieu de OC2A
  TCCR2A = _BV(COM2B1) | _BV(WGM21) | _BV(WGM20);  // Fast PWM, clear OC2B on match
  TCCR2B = _BV(CS20);  // Prescaler 1 → f_PWM = 16MHz / 256 = 62.5kHz
  OCR2B = 0;  // Duty cycle initial 0%
}

void loop() {
  
  static uint32_t lastUpdate = 0;
  
  // Gestion boutons
  handleButtons();
  
  // Génération modulation
  if (currentMode != MODE_BLANC) {
    generateModulation();
  } else {
    OCR2B = DUTY_BLANC;  // 50% duty cycle constant
  }
  
  // Mesure batterie toutes les 2 secondes
  if (millis() - lastUpdate > 2000) {
    lastUpdate = millis();
    uint16_t voltage = readBatteryVoltage();
    Serial.print(F("Batterie : "));
    Serial.print(voltage);
    Serial.print(F(" mV | Mode : "));
    Serial.print(modeNames[currentMode]);
    Serial.print(F(" | Freq : "));
    Serial.print(freqNames[currentFreq]);
    Serial.print(F(" ("));
    Serial.print(frequencies[currentFreq], 2);
    Serial.println(F(" Hz)"));
  }
}

void generateModulation() {
  
  uint32_t currentMicros = micros();
  
  // Calcul phase Nogier
  float nogierFreq = frequencies[currentFreq];
  float nogierPeriod = 1000000.0 / nogierFreq;
  float nogierPhase = (float)(currentMicros % (uint32_t)nogierPeriod) / nogierPeriod * 2.0 * PI;
  
  // Enveloppe sinusoïdale
  float envelope = 0.5 + 0.5 * sin(nogierPhase);
  
  // Modulation 70% profondeur, minimum 20%
  float modulatedIntensity = 0.20 + 0.70 * envelope;
  
  // Calcul duty cycle selon mode
  uint8_t baseDuty = (currentMode == MODE_DETECTION) ? DUTY_DETECTION : DUTY_TRAITEMENT;
  uint8_t finalDuty = (uint8_t)(baseDuty * modulatedIntensity);
  
  // Écriture PWM
  OCR2B = finalDuty;
}

void handleButtons() {
  
  static bool lastBtnFreq = HIGH;
  static bool lastBtnMode = HIGH;
  static uint32_t lastDebounceFreq = 0;
  static uint32_t lastDebounceMode = 0;
  
  bool btnFreq = digitalRead(BTN_FREQ_PIN);
  bool btnMode = digitalRead(BTN_MODE_PIN);
  
  // Bouton FREQ
  if (btnFreq != lastBtnFreq) lastDebounceFreq = millis();
  
  if ((millis() - lastDebounceFreq) > 50) {
    if (btnFreq == LOW && lastBtnFreq == HIGH) {
      currentFreq = (currentFreq + 1) % numFreqs;
      updateFreqIndicators();
      flashFeedback(2, 80);
      printStatus();
      delay(200);
    }
  }
  lastBtnFreq = btnFreq;
  
  // Bouton MODE
  if (btnMode != lastBtnMode) lastDebounceMode = millis();
  
  if ((millis() - lastDebounceMode) > 50) {
    if (btnMode == LOW && lastBtnMode == HIGH) {
      currentMode = (Mode)((currentMode + 1) % numModes);
      updateFreqIndicators();
      flashFeedback(currentMode + 1, 80);
      printStatus();
      delay(200);
    }
  }
  lastBtnMode = btnMode;
}

void updateFreqIndicators() {
  digitalWrite(LED_R_PIN, freqIndicators[currentFreq][0]);
  digitalWrite(LED_G_PIN, freqIndicators[currentFreq][1]);
  digitalWrite(LED_B_PIN, freqIndicators[currentFreq][2]);
}

void printStatus() {
  Serial.print(F("\n► Fréquence "));
  Serial.print(freqNames[currentFreq]);
  Serial.print(F(" ("));
  Serial.print(frequencies[currentFreq], 2);
  Serial.print(F(" Hz) | Mode : "));
  Serial.print(modeNames[currentMode]);
  Serial.print(F(" | Duty : "));
  if (currentMode == MODE_DETECTION) {
    Serial.print(DUTY_DETECTION);
    Serial.print(F(" (25% → ~5mA)"));
  } else if (currentMode == MODE_TRAITEMENT) {
    Serial.print(DUTY_TRAITEMENT);
    Serial.print(F(" (92% → ~20mA)"));
  } else {
    Serial.print(DUTY_BLANC);
    Serial.print(F(" (50% → ~10mA)"));
  }
  Serial.println();
}

void flashFeedback(uint8_t times, uint16_t delayMs) {
  uint8_t savedDuty = OCR2B;
  for (uint8_t i = 0; i < times; i++) {
    OCR2B = 255;
    delay(delayMs);
    OCR2B = 0;
    delay(delayMs);
  }
  OCR2B = savedDuty;
}

uint16_t readBatteryVoltage() {
  int adcValue = analogRead(ADC_BATTERY_PIN);
  // Conversion : 5V × (33k / 133k) = 1.24V max au nœud
  // ADC Uno : 10-bit (0-1023), Vref = 5V
  // Vbat = adcValue × (5000mV / 1023) × (133 / 33)
  uint16_t voltage = (uint32_t)adcValue * 5000 * 133 / 1023 / 33;
  return voltage;  // Tension en mV
}
```

═══ TEST 1 : LED PRINCIPALE (PWM simplifié) ═══

Objectif : Vérifier que la LED NSPW500CS s'allume et que le PWM fonctionne

Procédure :

 🔧 DÉPANNAGE (Troubleshooting)
═══════════════════════════════════════════════════════════════════════════════

PROBLÈME 1 : LED principale ne s'allume pas

  Vérifications :
    ☐ Polarité LED (anode + vers résistance, cathode - vers GND)
    ☐ Résistance 47Ω bien en série (vérifier avec multimètre)
    ☐ Code uploadé sans erreur
    ☐ Pin D3 configuré en OUTPUT
  
  Test manuel :
    • Connecter anode LED directement à 5V (via 47Ω)
    • Connecter cathode LED à GND
    • → Si LED s'allume, problème vient du PWM

PROBLÈME 2 : LED principale trop faible

  Cause probable : Résistance trop élevée ou duty cycle trop bas
  
  Solutions :
    • Vérifier résistance = 47Ω (pas 470Ω !)
    • Augmenter DUTY_TRAITEMENT de 235 à 255 (100%)
    • Mesurer tension aux bornes LED : devrait être 3.0-3.6V

PROBLÈME 3 : LED principale trop brillante / clignotement trop visible

  Cause probable : Modulation trop profonde ou fréquence PWM trop basse
  
  Solutions :
    • Réduire MODULATION_DEPTH dans generateModulation() de 0.70 à 0.50
    • Augmenter MIN_INTENSITY de 0.20 à 0.30
    • Vérifier fréquence Timer2 (devrait être > 10kHz)

PROBLÈME 4 : LED indicateurs ne s'allument pas

  Vérifications :
    ☐ Polarité LEDs (anode vers résistance, cathode vers GND)
    ☐ Résistances 470Ω présentes
    ☐ Pins D11, D10, D9 bien configurés en OUTPUT
  
  Test :
    • digitalWrite(LED_R_PIN, HIGH) dans setup()
    • LED Rouge doit s'allumer

PROBLÈME 5 : Boutons ne répondent pas

  Vérifications :
    ☐ Connexion bouton correcte (une patte vers Dx, autre vers GND)
    ☐ INPUT_PULLUP activé dans pinMode()
    ☐ Serial.println(digitalRead(BTN_FREQ_PIN)) affiche HIGH au repos
  
  Test anti-rebond :
    • Ajouter condensateur 100nF entre Dx et GND
    • Augmenter délai debounce de 50ms à 100ms

PROBLÈME 6 : Mesure batterie incorrecte

  Vérifications :
    ☐ Diviseur 100kΩ / 33kΩ bien connecté
    ☐ Nœud diviseur vers A0 (pas de court-circuit)
    ☐ Tension au nœud = Vbat × (33 / 133) (mesure multimètre)
  
  Calibration :
    • Mesurer Vbat avec multimètre : ex 5.00V
    • Mesurer Vnœud avec multimètre : devrait être 1.24V
    • Ajuster formule dans code si écart

PROBLÈME 7 : Consommation trop élevée

  Vérifications :
    ☐ Pas de court-circuit sur breadboard
    ☐ LED NSPW500CS (20mA), pas autre modèle haute puissance
    ☐ Résistance 47Ω (pas 4.7Ω !)
    ☐ PWM duty cycle correct (Serial Monitor affiche valeurs cohérentes)
  
  Mesure détaillée :
    • Déconnecter LED principale → Mesurer courant électronique seul (~8mA)
    • Reconnecter LED → Mesurer courant total (doit augmenter de 5-20mA selon mode)
