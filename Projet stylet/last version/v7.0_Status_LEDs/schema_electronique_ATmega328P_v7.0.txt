═══ DÉTAIL CONNEXION BOUTONS ═══

Tous les boutons utilisent pull-up interne (INPUT_PULLUP dans code).

 BTN_FREQ (Sélection fréquence Nogier) :
   PD2 ──┬──── [Bouton tactile] ──── GND
         │
    (Pull-up interne 20-50kΩ)
    Note : Interruption INT0 disponible pour détection asynchrone

 BTN_MODE (Sélection mode détection/traitement/blanc) :
   PD4 ──┬──── [Bouton tactile] ──── GND
         │
    (Pull-up interne 20-50kΩ)

 BTN_ON_OFF OPTIONNEL (Marche/Arrêt via RESET) :
   RESET ──┬──── Pull-up 10kΩ ──── VCC
           │
           └──── [Bouton] ──── GND
           │
       [C: 100nF]  (Anti-rebond)
           │
          GND

═══ DÉTAIL MESURE BATTERIE (ADC) ═══

  BAT+ (4.2V max)
      │
      └───[R1: 100kΩ]───┬──── PC0/ADC0 (A0)
                        │
                    [R2: 33kΩ]
                        │
                       GND

  Tension divisée = BAT+ × (33k / 133k) = BAT+ × 0.248
  
  Exemples :
    • BAT = 4.2V → ADC lit 1.04V
    • BAT = 3.7V → ADC lit 0.92V
    • BAT = 3.0V → ADC lit 0.74V

  Avec Vref = AVCC (3.3V) pour ATmega328P :
    ADC_value = (Vin / 3.3V) × 1023
    BAT_voltage = (ADC_value / 1023) × 3.3V / 0.248
    
  Alternative : Utiliser AREF interne 1.1V pour meilleure précision
    analogReference(INTERNAL) dans setup()
    ADC_value = (Vin / 1.1V) × 1023

═══ CALCUL RÉSISTANCE SÉRIE (R_LED) ═══

🎯 OBJECTIF : Limiter courant LED à 20mA (nominal) pour durée de vie maximale

Alimentation : 3.3V (régulateur LDO stable)
Vf LED       : 3.0V @ 20mA (typique)
If cible     : 20mA (mode TRAITEMENT max)

Calcul résistance :
  R = (Vsupply - Vf) / If
  R = (3.3V - 3.0V) / 0.020A
  R = 0.3V / 0.020A
  R = 15Ω

⚠️ PROBLÈME : 15Ω trop faible (courant élevé si Vf varie)

🎯 SOLUTION : Utiliser 47Ω pour sécurité + contrôle PWM pour ajuster intensité

Avec R = 47Ω :
  I_max = (3.3V - 3.0V) / 47Ω = 6.4mA à 100% duty cycle
  
  ✓ Sécurité garantie (impossible de dépasser 30mA max de la LED)
  ✓ Protection contre variations Vf (3.0-3.6V selon LED)
  ✓ Contrôle fin du courant via PWM duty cycle

═══ MODES DE FONCTIONNEMENT ET PWM ═══

Mode DÉTECTION (basse intensité - 5mA) :
  • Duty cycle PWM : 25% (baseDuty = 64/255)
  • Courant LED   : ~6.4mA × 0.25 × modulation_depth ≈ 5mA moyen
  • Consommation  : 5mA LED + 8mA système = 13mA total
  • Autonomie     : 900mAh / 13mA = 69 heures!

Mode TRAITEMENT (intensité maximale - 20mA) :
  • Duty cycle PWM : 92% (baseDuty = 235/255)
  • Courant LED   : ~20mA effectif (compense Vf + résistance)
  • Consommation  : 20mA LED + 8mA système = 28mA total
  • Autonomie     : 900mAh / 28mA = 32 heures!

Mode BLANC (continu non modulé - 10mA) :
  • Duty cycle PWM : 50% (baseDuty = 128/255)
  • Courant LED   : ~10mA constant
  • Consommation  : 10mA LED + 8mA système = 18mA total
  • Autonomie     : 900mAh / 18mA = 50 heures

🎉 AUTONOMIE MULTIPLIÉE PAR 5 à 13× vs version 6.0!

═══ MODULATION FRÉQUENCES NOGIER ═══

Identique à v6.0 :
  • Fréquences : 2.28Hz à 146Hz (7 fréquences A-G)
  • Porteuse   : 976Hz (Timer2, prescaler 64)
  • Profondeur : 70% (enveloppe sinusoïdale)
  • Intensité min : 20% (pas d'extinction totale)

La modulation est appliquée sur le duty cycle PWM de base :
  finalDuty = baseDuty × (MIN_INTENSITY + MODULATION_DEPTH × sin(phase))

Avec LED 20mA (vs 350mA v6), le clignotement est encore moins visible!

═══════════════════════════════════════════════════════════════════════════════
 6️⃣ LED INDICATEURS RGB (Inchangé vs v6.0)
═══════════════════════════════════════════════════════════════════════════════

Les LED indicateurs utilisent un expandeur I2C PCF8574A.

⚠️ ATTENTION : Le PCBA utilise un PCF8574A (variante "A"), dont l'adresse
  de base est 0x38 (avec A0=A1=A2=GND), PAS 0x20 comme le PCF8574 standard.
  Sur breadboard avec PCF8574 (sans A), l'adresse est 0x20.

⚠️ NOTE PCBA : Les 3 LEDs indicatrices sont toutes ROUGES (D1, D2, D3)
  et fonctionnent en ACTIVE-HIGH (écrire 1 = LED ON, 0 = LED OFF).
  Le codage des 7 fréquences utilise les combinaisons de 1 à 3 LEDs.

  ATmega328P PC4 (SDA) ───┐
  ATmega328P PC5 (SCL) ───┼─── [PCF8574A I2C] ──┬── LED D1 [470Ω] GND
                          │  (Adresse 0x38)    ├── LED D2 [470Ω] GND
                          │                     └── LED D3 [470Ω] GND
                          │
                     Pull-ups 4.7kΩ → VCC

Codage 7 fréquences Nogier (combinaisons de LEDs rouges) :
  • Freq A (2.28Hz)  : D1 seule          (1 LED)
  • Freq B (4.56Hz)  : D2 seule          (1 LED)
  • Freq C (9.12Hz)  : D1+D2             (2 LEDs)
  • Freq D (18.25Hz) : D3 seule          (1 LED)
  • Freq E (36.50Hz) : D1+D3             (2 LEDs)
  • Freq F (73Hz)    : D2+D3             (2 LEDs)
  • Freq G (146Hz)   : D1+D2+D3          (3 LEDs)

Indicateur batterie :
  • > 3.7V : Vert stable
  • < 3.5V : Rouge clignotant
  • < 3.3V : Coupure mode TRAITEMENT (sécurité)

Scénario 1 : MODE DÉTECTION (usage typique)

  Autonomie = 900mAh / 13mA = 69 heures (vs 26h en v6.0!)

Les points suivants ont été découverts lors du premier assemblage PCBA :

1. PCF8574A vs PCF8574 :
   Le PCBA utilise un PCF8574A (variante "A") avec adresse I2C de base 0x38
   (A0=A1=A2=GND). Le PCF8574 standard (sans "A") a une adresse de base 0x20.
   → Code flash version corrigé : PCF8574_ADDRESS = 0x38

2. LEDs Indicateurs ACTIVE-HIGH :
   Les 3 LEDs indicatrices (D1, D2, D3) sont toutes ROUGES et câblées en
   ACTIVE-HIGH : écrire 1 sur le port PCF8574A = LED allumée.
   → Code flash version corrigé : freqIndicators[] en active-HIGH

3. Oscillateur interne RC 8MHz :
   Le PCBA n'a PAS de cristal externe. L'ATmega328P fonctionne sur son
   oscillateur interne RC calibré à 8MHz (lfuse=0xE2).
   → Suffisamment précis pour Timer2 PWM et I2C 100kHz.

4. Fuses réels brûlés sur PCBA :
   lfuse = 0xE2 (oscillateur interne RC 8MHz, CKDIV8 désactivé)
   hfuse = 0xD9 (no bootloader, EESAVE off)
   efuse = 0xFF (BOD désactivé)

5. Programmation via USBasp (recommandé) :
   L'Arduino-as-ISP s'est révélé peu fiable (problèmes d'auto-reset).
   Le USBasp est fortement recommandé. Jumper JP1 (slow clock) nécessaire
   pour un chip neuf (1MHz par défaut avec CKDIV8 activé en usine).

6. VCC Pins TQFP-32 :
   Sur le package TQFP-32, les pins VCC sont 4 et 6 (PAS pin 7 comme
   en DIP-28). GND est sur pins 3, 5 et 21.
